# 支付与账务

支付与账务覆盖用户充值、余额变动、订单、套餐消费、返利、积分、VIP 和支付渠道配置。该部分属于高风险链路，上线必须使用真实回调 proof。

## 用户端能力

| 能力 | 说明 |
| --- | --- |
| 钱包余额 | 查看当前余额、冻结金额或可用余额。 |
| 在线充值 | 创建充值订单，跳转或展示支付渠道。 |
| 充值记录 | 查看充值状态、金额、渠道、订单号和完成时间。 |
| 消费记录 | 查看实例创建、续费、资源消费等账务流水。 |
| 订单中心 | 在 `/orders` 聚合查看充值订单和实例账单详情。 |
| 礼品卡 | 在 `/gift-cards` 使用余额生成礼品卡、兑换他人或后台发放的礼品卡，并查看自己的礼品卡记录。 |
| 兑换码 | 使用管理员发放的兑换码增加余额或权益。 |
| 推荐返利 | 查看邀请关系、返利记录和推广收益。 |

## 管理端能力

| 能力 | 说明 |
| --- | --- |
| 充值订单 | 查询订单、支付状态、回调结果和异常原因。 |
| 订单中心 | 在 `/admin/orders` 统一查看充值订单和实例账单，支持按用户、类型、状态、订单号、交易号和时间范围筛选、查看详情、处理充值异常、登记争议状态和发起退款/调账审批。 |
| 财务对账 | 在 `/admin/billing?tab=reconciliation` 按业务日汇总充值、余额流水、实例扣费、调账审批和托管收益，生成差异项和脱敏 CSV。 |
| 调账审批 | 订单退款、补款和扣款先生成审批任务，审批通过后才写入余额流水。 |
| 余额变动 | 审计用户余额增减、管理员调整和系统扣费。 |
| 支付渠道 | 配置支付提供商、商户号、密钥、回调地址和启用状态。 |
| 返利审核 | 审核推荐转化、返利结算和异常推广。 |
| VIP / 积分 | 管理会员等级、福利、积分变动和权益。 |
| 礼品卡 | 在 `/admin/gift-cards` 单张或批量生成礼品卡，查看统计和脱敏列表，并启用、禁用或删除未使用礼品卡。 |
| 兑换码 | 创建、禁用、使用记录和库存管理。 |

## 回调边界

- 支付回调地址应使用客户面板公网域名，例如 `https://panel.example.com/api/...`。
- 回调不应使用后台域名、内网后端地址或临时测试域名。
- 回调必须校验签名、订单号、金额、状态和幂等性。
- 支付密钥不得进入前端 bundle、日志或任务输出。

## 关键后端模块

- `recharge`：充值订单、支付回调和查询。
- `orders`：用户端和后台的统一订单中心，聚合充值订单与实例计费记录；后台页面复用充值人工完成、标记失败接口，记录订单争议状态，并为退款/补款/扣款创建调账审批。
- `balance`：余额变动、账务流水和调账审批；审批通过后才执行实际余额变更。
- `gift-cards`：礼品卡生成、兑换、余额锁、脱敏列表和后台管理。
- `admin-billing`：后台账务管理、日对账、差异处理和脱敏 CSV 导出。
- 支付渠道逻辑：渠道配置、密钥脱敏和出站请求。
- `aff`：推荐返利。
- `redeem-codes`：兑换码。
- `vip-levels`、`vip-benefits`：会员等级和权益。

## 风险点

- 回调重复投递必须幂等，不能重复加款。
- 金额必须以系统内部单位精确处理，避免浮点误差。
- 支付渠道密钥只能后端保存和使用。
- 管理员手动调账必须写审计日志，高风险退款/补款/扣款应先进入调账审批。
- 礼品卡属于余额类高风险能力，生产必须配置 `PAYINCUS_GIFT_CARD_ADMIN_IDS`，用户余额生成和兑换必须保持事务一致，后台列表默认脱敏。
- 订单中心的人工完成、失败标记和调账审批必须填写可追溯原因，不能绕过已有充值入账和余额流水逻辑。
- 退款登记只创建调账审批，不直接调用支付 provider 退款，也不直接修改余额。
- 订单详情只能展示脱敏 provider 摘要，不能返回回调原文、支付渠道配置快照或密钥。
- 财务对账导出只能包含必要业务字段，订单号和交易号等标识要脱敏，不能导出回调原文、支付渠道配置快照、密码、token 或密钥。
- 充值成功但通知失败时，账务状态仍应准确。

## 验证清单

- 真实支付下单成功。
- 真实支付回调成功入账。
- 重复回调不会重复入账。
- 错误签名、错误金额、错误订单号被拒绝。
- 后台能查到订单、回调结果和余额流水。
- 礼品卡用户余额生成会扣减余额并写入余额流水；兑换不能自兑，重复并发兑换只能成功一次。
- 后台礼品卡管理在生产未配置 `PAYINCUS_GIFT_CARD_ADMIN_IDS` 时拒绝访问，配置后列表默认脱敏。
- 用户端和后台订单中心只能返回各自权限范围内的订单，不暴露支付回调原始 payload 或支付渠道配置快照。
- 后台订单详情可以对待支付或处理中充值订单执行手动完成/标记失败，并可对关联用户提交带原因的调账审批。
- 后台订单详情可以登记争议状态：待核查、已确认、已补偿、已关闭。
- 同一订单已有待审核退款审批时，不能重复登记退款。
- 调账审批列表最多显示 7 条任务，更多任务分页查看；只有审批通过后才会产生余额日志。
- 日对账重复运行同一天不会重复写入差异项。
- 日对账差异项可以追踪来源、用户、金额、处理状态、处理人和备注。
- 财务 CSV 导出不包含敏感凭据、回调原文或支付渠道配置快照。
