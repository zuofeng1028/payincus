# 通知、工单与帮助

通知、工单和帮助中心负责客户沟通、运营公告、异常提醒和后台处理闭环。

## 用户端能力

| 能力 | 说明 |
| --- | --- |
| 工单 | 提交问题、上传附件、查看管理员回复、关闭工单。 |
| 通知中心 | 查看系统消息、账务提醒、资源提醒和运营通知。 |
| 帮助中心 | 阅读管理员维护的帮助文章和操作说明。 |
| 公告弹窗 | 接收站点公告、维护通知或重要变更提示。 |

## 管理端能力

| 能力 | 说明 |
| --- | --- |
| 工单处理 | 查看全部工单、回复、关闭、筛选和处理附件。 |
| 帮助文章 | 创建、编辑、发布或下线帮助内容。 |
| 公告广播 | 发布站内公告、弹窗公告和运营消息。 |
| 通知渠道 | 配置站内通知、邮件、Telegram 或其他投递渠道。 |
| 投递日志 | 查看通知发送结果、失败原因和重试状态。 |

## 投递渠道

- 站内信：默认通知渠道，用户在 `/inbox` 查看。
- SMTP：用于邮件验证码、工单提醒、账务提醒和系统通知。
- Telegram：用于管理员告警、用户绑定和关键事件通知。
- Lsky：用于工单附件或图片上传场景。

## 关键后端模块

- `tickets`：工单创建、回复、关闭和附件安全。
- `help`：帮助中心文章。
- `notifications`、`inbox`：通知创建、查询和投递记录。
- `admin-notification-channels`：后台通知渠道配置。
- `telegram`：Telegram webhook、绑定和通知。
- `system-config`：SMTP、附件、弹窗公告等配置。

## 风险点

- 工单附件必须限制大小、类型和访问权限。
- 通知投递失败不能阻塞核心账务或资源操作。
- Telegram webhook URL 和 token 不能泄露。
- SMTP 密码、Lsky token 等密钥必须脱敏。
- 公告弹窗不能影响用户端登录和支付回调。

## 验证清单

- 用户能创建工单并收到管理员回复。
- 管理员能筛选、回复和关闭工单。
- SMTP 真实投递成功。
- Telegram webhook 和通知投递成功。
- Lsky 上传、读取和权限控制正常。
