# 管理后台功能

管理后台面向管理员，只提供运营、配置、审核、交付和系统维护能力，不提供普通用户自助入口。

入口：

```text
https://admin.example.com
```

## 访问边界

- 后台登录页是 `/admin/login`。
- 旧 `/login` 仅兼容跳转到 `/admin/login`。
- 后台路由要求管理员身份，普通用户不能进入后台。
- 后台构建产物目录是 `client/dist/admin`。
- 后台 API 使用管理员鉴权，主要集中在 `/api/admin/*`。

## 运营总览

`/admin/statistics` 顶部提供商业运营总览，管理员可以先看当天经营状态，再进入用户、实例和计费趋势：

- 收入：今日、昨日、近 7 日和近 30 日已完成充值收入。
- 订单：今日订单总数、成功、失败、待处理和需要人工核对的支付订单。
- 用户：今日新增、今日活跃和历史付费用户。
- 实例：今日新购、当前运行、异常实例和 7 日内到期实例。
- 交付：待执行交付任务和 24 小时内失败交付任务。
- 基础设施：在线节点、在线 Agent、离线或心跳过期 Agent。
- 支持与通知：待处理工单、未读站内消息、24 小时通知投递失败。
- 风险提醒：支付渠道未启用、SMTP 未启用、通知通道缺失、节点或 Agent 离线、交付失败、支付异常、OTA 失败和 OTA 磁盘空间错误。

运营总览只通过管理员统计接口返回数据，不暴露到用户端 API。

## 容量与成本

`/admin/capacity-cost` 用于把资源余量和成本口径收口到一个后台运营页面：

- 汇总 Host CPU、内存、磁盘、NAT 端口、实例数量和流量使用，展示可售余量。
- 支持为每台 Host 维护月成本、IPv4 月成本、每 TB 流量成本和备注。
- 按套餐价格与 Host 成本估算月毛利，帮助识别低毛利或负毛利套餐。
- 容量预警只做运营提示，不自动停售、不修改套餐、不影响实例创建和计费。
- Host 容量压力会写入 SLA 告警事件，同类对象使用稳定指纹合并。
- 每天按 Host 写入容量快照，供 7/30 天趋势和后续运营复盘使用。
- 用户端套餐、实例和购买接口不返回 Host 成本、毛利或内部容量字段。

## 订单与支付运营

`/admin/orders` 支持把订单异常从发现、核查、补偿到关闭形成记录：

- 订单详情展示脱敏后的 provider 状态摘要，包括原始状态、渠道、脱敏交易号和回调时间，不展示回调原文或支付渠道配置快照。
- 争议状态包括“待核查、已确认、已补偿、已关闭”。
- 退款登记只创建调账审批任务，不会直接修改用户余额。
- 同一订单已有待审核退款审批时，后台会阻止重复登记。
- 退款、补款和扣款仍由调账审批执行，审批通过后才写入余额流水。
- 用户端订单中心不暴露后台运营处理、退款审批或 provider 内部摘要。

## 财务对账

`/admin/billing?tab=reconciliation` 提供日对账工作台：

- 按业务日生成一个对账任务，重复运行同一天会更新同一任务，不会重复写入差异项。
- 汇总成功充值、余额流水、实例扣费、已通过调账审批和托管收益。
- 自动检测成功充值未入账、关键余额流水缺少业务来源、付费实例已交付但无扣费记录、已通过调账审批但无余额流水。
- 差异项支持“差异、已确认、已忽略”状态和处理备注，保留处理人。
- CSV 导出覆盖订单、余额流水、托管收益和调账审批，导出内容不包含回调原文、支付密钥、token、密码或支付渠道配置快照。
- 读写权限已集中到财务对账权限入口，当前管理员可看可处理，后续可接入财务只读角色。

## 交付保障

`/admin/delivery` 用于处理付款后资源未交付、交付卡住或交付失败：

- 交付任务会按等待中、处理中、失败、成功展示，并自动识别失败任务和处理超过 30 分钟的疑似卡住任务。
- 保障状态包括“待人工处理、可自动重试、处理中、已恢复、已关闭”。
- 只有启动、停止、重启这类幂等任务允许重新入队；重装、重建、克隆、改节点等非幂等任务必须人工接管。
- 任务详情展示用户、实例、节点、Agent 心跳、节点资源、最近扣费记录和脱敏失败原因。
- 管理员可以人工接管、重新入队可重试任务、通知用户、标记恢复或关闭问题。
- 用户通知支持交付延迟、已恢复和需联系客服三种口径。
- 交付保障只记录处理备注、处理人、处理时间和动作历史，不返回 root 密码、节点证书、安装 token 或用户密码哈希。

## SLA 与告警

`/admin/sla-alerts` 用于把平台异常汇总成可处理的运营告警：

- 默认规则覆盖节点离线、Agent 离线、Agent 心跳过期、实例任务失败、充值订单失败、通知发送失败、邮件投递任务失败和 OTA 更新失败。
- 告警等级包括“提示、警告、严重”，状态包括“未处理、处理中、已恢复、已忽略”。
- 同一规则和同一对象使用稳定指纹合并，不会因为重复扫描无限生成记录。
- 管理员可以立即扫描、认领处理、标记恢复、忽略、静默告警和启停规则。
- 告警详情展示关联对象、触发次数、首次触发时间、最近触发时间、处理备注和动作历史。
- 告警内容只保存脱敏摘要和关联对象 ID，不保存支付回调原文、支付渠道配置快照、节点证书路径、安装 token、密码或密钥。
- 告警中心只在后台可见，用户端 API 和用户端构建产物不暴露告警规则、告警事件或内部处理备注。

## 工单与客服成功

`/admin/tickets` 提供客服工作台能力：

- 工单列表支持按活跃状态、工单来源和客服队列筛选；队列包括待处理、即将超时、已超时、等待用户和等待内部处理。
- 工单按优先级生成首响和解决 SLA，后台列表和详情页会显示 SLA 状态和截止时间。
- 后台工单详情聚合用户上下文，包括账户状态、脱敏邮箱、余额、最近订单、最近实例、最近交付保障 case 和最近 SLA 告警。
- 支持关联支付单、订单处理 case、实例、节点、交付保障 case、SLA 告警和插件任务，关联时会校验对象是否属于当前工单上下文。
- 内部备注与用户可见回复分开存储，内部备注只通过管理员接口读取，用户端工单详情和消息列表不会返回。
- 快捷动作只提供发送用户通知、打开调账申请、打开交付保障、打开用户/实例/节点页面，不直接修改余额或资源。
- 处理时间线合并用户回复、客服回复、内部备注和关联对象，方便客服复盘。
- 客服上下文不返回支付回调原文、支付渠道配置快照、IP、User-Agent、实例 root 密码、2FA secret、token、证书或其他密钥。

## 用户生命周期

`/admin/user-lifecycle` 用于把用户运营动作从临时人工查询收口成可追踪工作台：

- 生命周期总览显示用户总数、活跃用户、即将到期实例、标签统计、分群统计和近期运营动作。
- 标签覆盖新用户、已付费、高价值、即将到期、流失风险和异常风险；添加和移除标签都会写入运营动作记录。
- 默认分群覆盖新注册、已付费、高价值、即将到期和流失风险，管理员可以刷新分群结果。
- 用户列表支持按标签、分群、累计充值、实例数量和活跃状态筛选。
- 用户摘要展示脱敏邮箱、累计充值、累计消费、实例数量、最早到期、工单数量、生命周期事件和运营动作。
- 管理员可以为单个用户发放定向资源兑换码，兑换码会绑定 `target_user_id`，其他用户无法使用。
- 批量提醒必须先选择用户并勾选确认，提醒通过站内信发送并写入运营动作记录。
- 普通用户端只展示当前用户可用的定向兑换码，不显示后台标签、分群、内部事件或运营动作记录。
- 生命周期接口不返回支付回调原文、支付渠道配置快照、IP、User-Agent、实例 root 密码、2FA secret、token 或密钥。

## 日志与审计

`/admin/logs` 用于追踪后台操作、系统事件和高风险动作：

- 日志列表会展示风险等级，包括低、中、高和严重。
- 高风险操作目录覆盖系统更新、支付通道、余额调账、批量删除实例、节点、套餐、管理员权限和插件安装等动作。
- 日志页顶部展示风险动作数量、本页高风险、需要审批和需要验证码的记录数。
- 管理员可以按当前筛选导出审计 CSV，单次最多导出 1000 条。
- 审计导出会脱敏邮箱、IP、token、JWT 类内容，不导出密码、证书、完整支付回调或密钥。
- 每次审计导出都会写入 `audit.export` 操作日志。
- 风险目录和全站审计导出只对管理员开放，普通用户只能读取自己范围内的日志。

## 系统配置

后台设置拆成多个配置页：

- 访问与注册：注册开关、邀请码、登录策略。
- 托管与站点：托管资源、站点地址、客户注册链接基础 URL。
- 品牌与外观：站点名称、Logo、主题展示。
- 安全验证：敏感操作验证码、登录安全、密钥相关配置。
- 邮件服务：SMTP、发信模板和投递设置。
- 工单与附件：附件大小、上传安全和 Lsky 存储。
- 弹窗公告：用户端弹窗内容和展示策略。

## 版本更新

`/admin/system-update` 用于后台 OTA：

- 查看当前版本、Git tag、commit、构建时间和部署时间。
- 检查最新版本和 release tag 更新内容；当前已是最新版本时仍展示最新版本，并显示“已更新至最新版本”。
- 启动在线更新任务并查看实时日志。
- 更新任务列表最多显示 7 条，更多历史任务通过分页查看。
- 更新失败时自动回滚，成功后支持手动回滚到上一版。
- 支持 Release artifact、SHA256 校验和 `current` / `releases` 原子切换。
- 更新和回滚会保留 `plugins`、`plugin-data`、`plugin-logs` 和 `plugin-staging` 插件运行目录。

## 验证重点

- 管理员可以登录后台，普通用户会被拦截。
- 后台不出现用户端自助充值、转移、签到、好友等客户功能入口。
- 后台生成客户链接时使用客户域名，不使用后台域名。
- 交付保障只暴露任务、实例、用户、节点、Agent、扣费和处理状态上下文，不返回 root 密码、证书、Token 或用户密码哈希；非幂等交付任务不能被自动重试。
- SLA 与告警必须保持 admin-only，同类告警必须合并，告警内容必须脱敏，用户端不能出现告警中心入口。
- 容量与成本必须保持 admin-only；成本配置、毛利估算和内部容量快照不能出现在用户端 API 或用户端构建产物中，且不能改变实例创建、套餐销售和计费行为。
- 工单客服上下文必须保持 admin-only，内部备注不能出现在用户端接口，快捷动作不能绕过调账审批或交付保障流程。
- 用户生命周期中心必须保持 admin-only；普通用户只能读取自己的可用定向兑换码，不能读取标签、分群、内部事件或其他用户运营动作。
- 日志与审计必须保持 admin-only；风险动作目录和审计 CSV 导出不能暴露给用户端，导出内容必须脱敏并记录导出日志。
- 订单中心只返回脱敏 provider 摘要，不返回支付回调原文、支付渠道配置快照或密钥。
- 财务对账导出必须保持脱敏，不返回回调原文、支付渠道配置快照、密码、token 或密钥。
- 后台 OTA 更新前后都要执行 split host、边界守卫和健康检查。
